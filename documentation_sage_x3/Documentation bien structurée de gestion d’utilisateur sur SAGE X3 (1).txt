Documentation bien structurée de gestion d’utilisateur sur SAGE X3
Introduction:
Cette documentation fournit une description complète et structurée du processus de création d’un utilisateur dans SAGE X3, en mettant l’accent sur les différences entre un utilisateur Syracuse (authentification web) et un utilisateur X3 (droits d’accès à l’ERP). Elle détaille les onglets, les champs, les boutons, et l’importance des profils de menu et fonctionnels, tout en étant adaptée pour une utilisation avec un modèle RAG (Retrieval-Augmented Generation) pour automatiser la gestion des utilisateurs. Les informations sont basées sur des ressources fiables et des pratiques standard de SAGE X3.


1 . PREMIERE ETAPE : CREATION D'UN UTILISATEUR
1. 1 Introduction
La création d’un utilisateur dans SAGE X3 est une étape fondamentale pour permettre aux employés ou partenaires d’accéder aux fonctionnalités de l’ERP. Ce processus comprend deux étapes distinctes : la création d’un utilisateur Syracuse pour l’authentification via l’interface web, et la création d’un utilisateur X3 pour définir les autorisations dans l’ERP. Cette section explique chaque étape, les onglets impliqués, les champs à remplir, et les bonnes pratiques pour garantir une configuration sécurisée et efficace.
Objectifs
* Créer un utilisateur Syracuse pour l’accès à l’interface web.
* Configurer un utilisateur X3 avec les droits et l’interface appropriés.
* Fournir une base pour automatiser ce processus via un modèle RAG.
Prérequis
* Privilèges administratifs dans SAGE X3.
* Accès aux fonctions Administration > Administration > Users > Users et Paramétrage > Utilisateurs > Utilisateurs (GESAUS).
* Connaissance des profils de menu et fonctionnels disponibles.
2. Différence entre utilisateur Syracuse et utilisateur X3
* Utilisateur Syracuse : Cet utilisateur est créé pour gérer l’authentification initiale via le serveur Syracuse, qui est la plateforme web de SAGE X3. Il permet aux utilisateurs de se connecter à l’interface via un navigateur web. Les paramètres sont configurés dans Administration > Administration > Users > Users, et incluent le login, le mot de passe, et l’appartenance à un groupe.L’utilisateur Syracuse est requis pour l’authentification initiale, permettant l’accès à l’interface web. La création d’un utilisateur Syracuse est la première étape pour permettre l’accès au système.
* Utilisateur X3 : Cet utilisateur définit les permissions et les droits d’accès dans l’ERP SAGE X3. Il est configuré via la fonction GESAUS (Paramétrage > Utilisateurs > Utilisateurs) et est lié à l’utilisateur Syracuse par le champ login. Il inclut des configurations comme les profils de menu, les profils fonctionnels, et les codes d’accès. L’utilisateur X3 détermine ce que l’utilisateur peut faire dans l’ERP, comme accéder à des modules spécifiques (Ventes, Achats) ou effectuer des actions (consultation, modification).
3. Création d’un utilisateur Syracuse:
3.1 Accès à la fonction
* Navigation : Connectez-vous à SAGE X3 avec des privilèges administratifs et accédez à Administration > Administration > Users > Users.
* Cette section gère les utilisateurs web pour l’authentification Syracuse.
3.2 Création du compte
* Cliquez sur le bouton Nouveau (ou « New User » selon la langue de l’interface).
   * Remplissez les champs suivants :
Dans Sage X3, la gestion des utilisateurs repose sur plusieurs champs qui définissent leur identité, leurs droits d'accès et leurs préférences d'interface. Le champ Code est un identifiant unique pour chaque utilisateur, composé de 5 caractères alphanumériques en majuscules (par exemple, "fben  foulen"), utilisé dans tout le système pour référencer l'utilisateur. Le champ Login indique le nom utilisé par l'utilisateur pour se connecter, lié à l'utilisateur Syracuse pour l'authentification (par exemple, "fben  foulen"). Le champ Actif est une case à cocher qui détermine si le compte est actif : s'il est coché, l'utilisateur peut se connecter ; sinon, le compte est désactivé. Le champ Connexion X3, lorsqu'il est coché, autorise l'accès au système ERP, tandis que le champ Connexion aux services web, s'il est coché, permet l'accès aux services web. Pour personnaliser l'expérience utilisateur, le champ Profil de menu définit le profil de menu, qui adapte l'interface en affichant uniquement les fonctions pertinentes pour le rôle de l'utilisateur (par exemple, "Menu Admin Ventes"). De même, le champ Profil fonctionnel attribue un profil fonctionnel qui contrôle les droits d'accès aux fonctions spécifiques de l'ERP, comme consulter, modifier ou exécuter certaines tâches (par exemple, "Fonction Admin Ventes"). Le champ Langue configure la langue de l'interface pour correspondre aux préférences de l'utilisateur (par exemple, "Français"), et le champ Email enregistre l'adresse email pour les notifications ou la communication (par exemple, "flen.ben foulen@entreprise.com"). En option, le champ Photo permet de télécharger une photo de profil, bien que cela ne soit pas obligatoire.
   * D'autres champs gèrent la sécurité et les informations supplémentaires. Le champ Nouveau mot de passe sert à définir un mot de passe initial, et la case Changement de mot de passe requis, si cochée, oblige l'utilisateur à modifier son mot de passe lors de sa première connexion pour renforcer la sécurité. L'option Mot de passe sans expiration, si activée, permet au mot de passe de ne jamais expirer, bien que cela soit déconseillé pour des raisons de sécurité. Le champ Authentification précise la méthode d'authentification, comme "Standard" ou "LDAP", selon la configuration de l'organisation. Si l'authentification LDAP est utilisée, le champ Instance LDAP indique l'instance LDAP à employer pour la vérification. Les champs Titre (par exemple, "M." ou "Mme"), Prénom et Nom stockent respectivement le titre, le prénom et le nom de l'utilisateur. Le champ Groupe affecte l'utilisateur à un groupe spécifique, qui peut définir des droits d'accès plus larges ou des rôles organisationnels (par exemple, "Utilisateur Basique"). Enfin, le champ Rôles permet d'assigner des rôles hiérarchiques ou des filtres de données, comme restreindre l'accès à certains ensembles de données (par exemple, rôle "ORD" pour les données liées aux commandes). Ces champs garantissent que chaque utilisateur dispose d'un accès et d'une expérience adaptés à ses responsabilités et aux politiques de sécurité de l'organisation dans Sage X3.
* 
Boutons :
   * Plus (+) : Ouvre la liste des groupes pour assigner l’utilisateur à un groupe (ex. : « Basic User »).
   * OK : Confirme la sélection du groupe.
   * Enregistrer : Sauvegarde l’utilisateur Syracuse.
3.3 Assignation à un groupe
   * Cliquez sur l’icône Plus (+) pour ouvrir la liste des groupes.
   * Cochez le groupe approprié (ex. : « Basic User ») et cliquez sur OK.
   * Les groupes définissent les droits d’accès initiaux et les endpoints accessibles.
3.4 Exemple
Pour créer un utilisateur Syracuse nommé « flen ben foulen » :
   * Login : fben  foulen
   * Actif : Coché
   * Authentification : Standard
   * Mot de passe : Temp123
   * Changement de mot de passe requis : Coché
   * Prénom : flen
   * Nom : ben foulen
   * Email : flen.ben foulen@entreprise.com
   * Groupe : Basic User
   * Cliquez sur Enregistrer.
4. Création d’un utilisateur X3
4.1 Accès à la fonction
   * Navigation : Accédez à Paramétrage > Utilisateurs > Utilisateurs (fonction GESAUS).
   * Cette fonction permet de configurer les utilisateurs X3 et leurs droits d’accès.
4.2 Création du profil utilisateur
   * Cliquez sur Nouveau pour créer un nouveau profil.
   * Remplissez les champs dans l’onglet Général :
Dans Sage X3, lors de la création d'un utilisateur via la fonction Paramétrage > Utilisateurs > Utilisateurs (GESAUS), l'onglet Général regroupe des champs essentiels pour définir l’identité, le statut, les accès et les préférences de l’utilisateur. Le champ Code est un identifiant unique de 5 caractères en majuscules, comme "FBENF" pour un utilisateur nommé Flen Ben Foulen, permettant de le repérer distinctement dans le système. Le champ Nom indique le nom complet de l’utilisateur, par exemple "Flen Ben Foulen", facilitant son identification dans les interfaces et les rapports.
4.2.1 Statut et accès au système
Le champ Actif, une case à cocher, doit être coché pour activer le profil de l’utilisateur ; sinon, le compte reste inactif et l’accès au système est bloqué. Le champ Connexion X3 (USRCONNECT), lorsqu’il est coché, autorise l’utilisateur à se connecter à l’ERP Sage X3, une condition indispensable pour travailler dans le système. À ce jour, Sage X3 distingue différents niveaux de "collaborateurs" grâce à ce champ, créant deux catégories : les utilisateurs pouvant se connecter au système et les collaborateurs sans accès. De même, le champ Connexion aux services web (USRCONXTD), activé selon les besoins, permet de se connecter à Sage X3 pour exécuter des Web-services, utile pour les intégrations ou les tâches automatisées. Le champ Pas de contrôle annuaire (WITHOUTLDA), une option configurable, désactive la vérification LDAP si l’authentification ne repose pas sur un annuaire externe.
4.2.2 Identification et profils
Le champ Login (LOGIN) relie l’utilisateur à son compte Syracuse (interface web) en spécifiant un identifiant de connexion système, tel que "fbenfoulen". Saisi dans la boîte de connexion à l’application, ce login, qui peut être plus long que le code interne limité à 5 caractères, doit être unique pour chaque utilisateur. Par défaut, il reprend le code de l’utilisateur, mais il peut être personnalisé. Le champ Code métier (CODMET) associe l’utilisateur à un profil métier, regroupant des utilisateurs partageant les mêmes profil de menu, profil fonctionnel et, éventuellement, profils BI ou Safe X3 WAS, pour une gestion cohérente des droits.
4.2.3 Paramètres opérationnels
Le champ Dépôt par défaut (WRH) sélectionne le dépôt principal associé à l’utilisateur pour les transactions de stocks, achats, ventes et production, comme "DEP01" pour un dépôt spécifique. Non obligatoire, ce champ prend la priorité sur le dépôt défini au niveau du site lorsqu’il est renseigné. Le champ Profil de menu (PRFMEN), ou champ DEFMEN, détermine l’arborescence de menu proposée par défaut à l’utilisateur lors de sa connexion, par exemple "Sales Admin Menu" pour un administrateur des ventes, sans toutefois définir les habilitations. Ces dernières sont gérées par le champ Profil fonctionnel (PRFFCT), ou champ DEFFCT, qui précise les droits d’accès aux fonctionnalités de l’ERP, comme consulter, modifier ou exécuter, en fonction du rôle, par exemple "Sales Admin Function". Ces habilitations sont complétées par les rôles, les paramètres utilisateur et les codes d’accès définis dans d’autres onglets.
4.2.4 Profils complémentaires et communication
Le champ Profil utilisateur BI (USRBI), ou champ DEFBI, attribue des droits pour les rapports Business Objects, permettant à l’utilisateur de visualiser ou modifier des états BI sous un code commun, comme "BI_SALES". Le champ Profil utilisateur GED (ARCPRF) gère les droits pour la gestion électronique des documents, tandis que le champ Profil Safe X3 WAS (PRFXTD), ou champ DEFXTD, définit les autorisations pour les services web avancés. Pour la communication, le champ Adresse mail Workflow (ADDEML) enregistre l’email de l’utilisateur, par exemple "flen.benfoulen@entreprise.com", utilisé par le système de workflow intégré pour envoyer des notifications. Les champs Téléphone par défaut (TELEP) et Fax par défaut (FAX1) fournissent les coordonnées de contact, comme "01 23 45 67 89" pour le téléphone.
4.2.5 Sécurité et personnalisation
Le champ Accès (ACSUSR) utilise un code d’accès pour restreindre la visualisation ou la modification de la fiche utilisateur à certains utilisateurs autorisés. Si ce champ est rempli, seuls ceux ayant des droits de lecture peuvent consulter l’enregistrement, et seuls ceux ayant des droits d’écriture peuvent le modifier. Le champ Destinations utilisateur (USRPRT) permet d’adopter les destinations d’impression d’un autre utilisateur, simplifiant la configuration. Le tableau Destinations par défaut, avec le champ Type (TYPIMP) et le champ Destination (PRTDEF), définit les imprimantes par type (basé sur le menu local paramétrable 22) et leurs destinations par défaut, comme "PDF" pour une impression numérique. Enfin, le champ Photo (AUSPIC), de type image, permet d’ajouter une photographie de l’utilisateur pour personnaliser son profil.
Une configuration précise de ces champs dans l’onglet Général est cruciale pour garantir que l’utilisateur dispose des accès nécessaires, d’une interface adaptée et de paramètres correspondant à ses responsabilités dans Sage X3.


Boutons :
   * Nouveau : Crée un nouveau profil.
   * Enregistrer : Sauvegarde le profil.
   * Annuler : Annule les modifications.
CREATION DES UTILISATEURS: 
Pour créer un utilisateur dans l'ERP Sage X3, vous devez suivre un processus en deux étapes : d'abord créer un utilisateur Syracuse (authentification web), puis configurer un profil utilisateur X3 (droits d'accès à l'ERP). Voici une description claire et concise du processus, avec les champs à remplir et les onglets correspondants, organisée dans l'ordre chronologique pour une utilisation dans un fichier JSON à envoyer à l'API de Sage X3.
Points clés
   * Le processus implique deux couches : Syracuse pour l'authentification et X3 pour les autorisations ERP.
   * Les champs doivent être remplis dans un ordre précis, d'abord pour Syracuse, puis pour X3.
   * Les informations sont extraites de sources fiables, mais certains champs peuvent varier légèrement selon la configuration de votre instance Sage X3.
Étape 1 : Création de l'utilisateur Syracuse
Navigation : Administration > Administration > Utilisateurs > Utilisateurs

Description : Cette étape configure l'utilisateur web pour l'authentification via Syracuse.
Champs à remplir :
   * Login : Nom d'utilisateur pour la connexion (par exemple, "jdupont").
   * Actif : Cocher pour activer l'utilisateur.
   * Authentification : Sélectionner la politique (par exemple, "Standard").
   * Nouveau mot de passe : Définir un mot de passe initial.
   * Mot de passe sans expiration : Option pour un mot de passe permanent.
   * Changement de mot de passe requis : Forcer le changement à la première connexion.
   * Titre : Titre de l'utilisateur (par exemple, "M." ou "Mme").
   * Prénom : Prénom de l'utilisateur (par exemple, "Jean").
   * Nom : Nom de famille (par exemple, "Dupont").
   * Email : Adresse email (par exemple, "jean.dupont@entreprise.com").
   * Groupe : Assigner à un groupe (par exemple, "Utilisateur de base").
Étape 2 : Création du profil utilisateur X3
Navigation : Paramétrage > Utilisateurs > Utilisateurs

Description : Cette étape définit les droits d'accès dans l'ERP X3.
Champs à remplir :
   * Code : Identifiant unique, souvent identique au login (par exemple, "JDUPONT").
   * Nom : Nom complet (par exemple, "Jean Dupont").
   * Actif : Cocher pour activer le profil.
   * Connexion X3 : Cocher pour autoriser l'accès à l'ERP.
   * Connexion aux services web : Cocher si nécessaire.
   * Login : Sélectionner le login Syracuse créé précédemment.
   * Profil de menu : Choisir un profil de menu (par exemple, "Menu Admin Ventes").
   * Profil fonctionnel : Choisir un profil fonctionnel (par exemple, "Fonction Admin Ventes").
   * Paramètres : Configurer les paramètres spécifiques (par exemple, limites de sessions).
   * Accès : Définir les codes d'accès ou cocher "Tous les codes d'accès".
exemple:
json


{
  "syracuse_user": {
    "login": "jdupont",
    "active": true,
    "authentication": "Standard",
    "new_password": "Temp123",
    "password_never_expires": false,
    "require_password_change": true,
    "title": "M.",
    "first_name": "Jean",
    "last_name": "Dupont",
    "email": "jean.dupont@entreprise.com",
    "group": "Basic User"
  },
  "x3_user": {
    "code": "JDUPONT",
    "name": "Jean Dupont",
    "active": true,
    "x3_connection": true,
    "web_services_connection": false,
    "login": "jdupont",
    "menu_profile": "Sales Admin Menu",
    "function_profile": "Sales Admin Function",
    "parameters": {},
    "access": ["All access codes"]
  }
}

Documentation Détaillée des Habilitations Fonctionnelles :
Documentation détaillée des habilitations fonctionnelles dans Sage X3
Les habilitations fonctionnelles dans Sage X3 constituent un pilier central du système de sécurité de l’ERP, permettant de contrôler avec précision les actions qu’un utilisateur ou un groupe d’utilisateurs peut effectuer. Elles sont gérées via des profils fonctionnels, qui définissent les droits d’accès aux fonctions, aux données et aux processus, indépendamment de l’interface utilisateur visible (contrôlée par les profils de menu). Cette documentation, extraite de sources fiables, explique leur rôle, leur utilité, les cas où elles sont nécessaires, et le processus détaillé de configuration, avec les champs à remplir dans l’ordre chronologique.
Rôle des habilitations fonctionnelles
Les habilitations fonctionnelles déterminent les permissions d’un utilisateur pour chaque fonction de Sage X3, comme consulter, créer, modifier, supprimer ou exécuter des actions spécifiques. Elles sont essentielles pour :
   * Sécuriser l’accès : Limiter les utilisateurs aux fonctionnalités nécessaires à leurs rôles, suivant le principe du moindre privilège.
   * Personnaliser les droits : Adapter les permissions selon les besoins organisationnels, par exemple en restreignant la modification des données financières.
   * Gérer les restrictions par site : Contrôler les accès en fonction des sites ou groupes de sites, par exemple pour limiter un utilisateur à un site spécifique comme "Paris".
Elles sont distinctes des profils de menu, qui contrôlent l’interface visible, et des codes d’accès, qui protègent des données spécifiques.
Quand utiliser les habilitations fonctionnelles ?
Les habilitations fonctionnelles sont nécessaires dans les cas suivants :
   * Sécurité des données sensibles : Par exemple, empêcher un utilisateur de supprimer des commandes ou d’accéder aux données RH.
   * Conformité réglementaire : Respecter les politiques de sécurité en limitant les permissions aux utilisateurs autorisés.
   * Gestion multi-site : Configurer des droits différents selon les sites ou groupes, comme accorder un accès complet à un module sauf pour un site particulier.
   * Personnalisation des rôles : Adapter les permissions pour des rôles spécifiques, comme un gestionnaire RH qui peut consulter mais pas modifier certaines données.
Elles sont particulièrement utiles dans les environnements complexes avec de nombreux utilisateurs, sites ou modules.
Comment utiliser les habilitations fonctionnelles ?
Les habilitations fonctionnelles sont configurées via la fonction GESAFT (Paramétrage > Utilisateurs > Profils fonctionnels) dans Sage X3. Le processus consiste à :
   1. Créer ou sélectionner un profil fonctionnel.
   2. Associer ce profil à des fonctions spécifiques (par exemple, "GES_RH" pour la gestion RH).
   3. Définir les droits d’accès (lecture, modification, etc.) et les restrictions par site ou groupe.
   4. Enregistrer les modifications et mettre à jour les tables associées.
Les droits sont spécifiés à l’aide d’options comme :
   * C : Création de fiches.
   * M : Modification de fiches.
   * S : Suppression de fiches.
   * V : Validation d’actions.
   * P : Impression de rapports.
Ces options varient selon la fonction, et leur documentation est disponible dans les prérequis de chaque fonction.
Processus de paramétrage des habilitations fonctionnelles
Le paramétrage des habilitations fonctionnelles suit un processus structuré, impliquant plusieurs étapes et champs à remplir dans un ordre chronologique. Voici le détail :
Étape 1 : Accéder à l’écran de configuration
   * Navigation : Paramétrage > Utilisateurs > Profils fonctionnels (GESAFT).
   * Prérequis : Privilèges administratifs pour accéder à la fonction.
   * Action : Ouvrir l’écran pour créer ou modifier un profil fonctionnel.
Étape 2 : Créer ou sélectionner un profil fonctionnel
   * Champ : Code profil (PRFCOD)
   * Description : Identifiant unique du profil (par exemple, "RH_MANAGER").
   * Requis : Oui.
   * Étape : Saisi en premier lors de la création du profil.
   * Champ : Description
   * Description : Nom descriptif du profil (par exemple, "Manager Ressources Humaines").
   * Requis : Non, mais recommandé.
   * Étape : Saisi immédiatement après le code profil.
Étape 3 : Associer des fonctions au profil
   * Champ : Fonction (FNC)
   * Description : Code de la fonction à autoriser (par exemple, "GES_RH" pour la gestion RH).
   * Requis : Oui.
   * Étape : Ajouté pour chaque fonction que le profil doit contrôler.
   * Champ : Module
   * Description : Module associé à la fonction (par exemple, "RH" pour Ressources Humaines).
   * Requis : Non, mais automatiquement déduit du code fonction.
   * Étape : Vérifié ou saisi lors de l’ajout de la fonction.
Étape 4 : Configurer les autorisations pour chaque fonction
Pour chaque fonction, vous pouvez définir plusieurs lignes d’autorisation, notamment pour des restrictions par site ou groupe. Les champs suivants sont remplis dans l’ordre :
   * Champ : Type (FCYGRUCOD)
   * Description : Type d’habilitation ("S" pour site, "G" pour groupe, vide pour par défaut).
   * Requis : Oui.
   * Étape : Saisi pour chaque ligne d’autorisation.
   * Champ : Regroupement/Site (FCYGRU)
   * Description : Code du site (ex. : "PARIS") ou du groupe (ex. : "EUROPE_NORD").
   * Requis : Oui si Type est "S" ou "G", sinon vide.
   * Étape : Saisi après le Type.
   * Champ : Accès (ACS)
   * Description : Autorisation ("Oui" pour permettre, "Non" pour refuser).
   * Requis : Oui.
   * Étape : Saisi après le Regroupement/Site.
   * Champ : Options (OPT)
   * Description : Droits spécifiques (ex. : "CMVS" pour Création, Modification, Validation, Suppression).
   * Requis : Non, dépend de la fonction.
   * Étape : Saisi en dernier pour chaque ligne.
Étape 5 : Enregistrer et mettre à jour
   * Action : Enregistrer les modifications.
   * Mise à jour de la table AFCTFCY : Si le système le demande, confirmer la mise à jour de la table des croisements d’habilitation (AFCTFCY). Pour les environnements avec de nombreux sites, utiliser l’utilitaire "Validation des fonctions" pour différer cette mise à jour.
Exemple de configuration
Scénario : Accorder au profil "RH_MANAGER" un accès complet au module RH, sauf pour le site "LYON" où seul un accès en lecture est permis.
   * Profil : PRFCOD = "RH_MANAGER", Description = "Manager Ressources Humaines".
   * Fonction : FNC = "GES_RH".
   * Autorisations :
   * Ligne 1 : FCYGRUCOD = vide, FCYGRU = vide, ACS = "Oui", OPT = "CMVS" (accès complet par défaut).
   * Ligne 2 : FCYGRUCOD = "S", FCYGRU = "LYON", ACS = "Oui", OPT = "" (lecture seule, pas d’actions supplémentaires).








Documentation du Fonctionnement des Codes d'Accès
Les codes d'accès dans Sage X3 sont des identifiants uniques qui déterminent les permissions accordées aux utilisateurs sur des fonctions, données ou processus spécifiques. Chaque code est associé à des droits précis (Lecture, Écriture, Exécution) et peut être attribué :
Directement à un utilisateur via son profil
Indirectement via des profils fonctionnels ou des règles métier
Exemple Typique :
Un code FIN_APPROVAL peut autoriser la validation des paiements, tandis que HR_READONLY restreint l’accès aux données RH à la consultation seule.
2. Interfaces Clés pour la Gestion des Accès
A. Écran Principal des Codes d’Accès (GESACS)
Chemin d’accès :
Superviseur → Sécurité → Codes d’accès
Fonctionnalités Principales :
Création/Modification :
Le champ CODACC définit l’identifiant du code (ex: INV_MODIFY pour la modification des inventaires).
Le champ DESACC décrit son usage (ex: "Permet de corriger les niveaux de stock").
Attribution Granulaire :
Les codes peuvent être liés à des objets spécifiques (écrans, transactions, rapports) pour un contrôle précis.
Actions Importantes :
La fonction Copie Inter-Dossiers permet de répliquer une configuration de codes vers d’autres environnements (ex: de la production vers un dossier de test).
L’intégration avec LDAP/Active Directory est possible pour synchroniser les droits avec des groupes externes.
B. Onglet "Accès" dans la Fiche Utilisateur
Chemin d’accès :
Administration → Utilisateurs → [Fiche Utilisateur] → Onglet Accès
Droits Configurables :
ACSUSR : Code d’accès global pour restreindre la modification de la fiche utilisateur elle-même.
Permissions par Objet :
R (Lecture) : Accès aux données sans modification.
W (Écriture) : Droit de modifier des champs ou des enregistrements.
X (Exécution) : Autorisation de lancer des traitements ou des workflows.
Exemple :
Un utilisateur avec ACSUSR = FIN_AUDIT et W activé pour le module COMPTA peut éditer des écritures comptables, mais pas les valider (nécessite X).
C. Table des Habilitations Fonctionnelles (AFCTFCY)
Accès via : GESAFT (Gestion des Profils Fonctionnels)
Mécanismes Clés :
FCYGRUCOD :
Valeur S pour une restriction par Site.
Valeur G pour une restriction par Groupe de Sites.
OPT (Options Complémentaires) :
V : Droit de validation (ex: approuver un bon de commande).
P : Permission de changer son mot de passe.
A : Accès aux paramètres avancés (ex: modification des codes d’accès).
ACS (Accès) :
A (Autoriser) ou R (Refuser) pour un site/groupe donné.
Cas d’Usage :
Pour un profil LOGISTICS_MANAGER, on peut autoriser (A) la modification des stocks (OPT = M) sur tous les sites sauf LYON (où ACS = R).
3. Bonnes Pratiques pour une Gestion Optimale
Hiérarchisation des Droits :
Utilisez des codes génériques (ex: FIN_VIEW) pour des droits larges.
Créez des codes spécifiques (ex: PAY_APPROVE_OVER_10K) pour des actions sensibles.
Audit et Supervision :
Consultez régulièrement la table AESPION pour tracer les modifications de permissions.
Lancez l’utilitaire Validation des Fonctions pour vérifier l’intégrité des habilitations.
Délégation Contrôlée :
Évitez d’activer la case "Tous codes d’accès" sauf pour les super-administrateurs.
Utilisez le paramètre AUZFCT pour limiter les profils attribuables par rôle.
4. Erreurs Courantes et Résolutions
Conflits de Sites :
Si un site appartient à deux groupes avec des droits incompatibles, Sage X3 bloque la saisie. Vérifiez les affiliations dans GESSIT.
Options Invalides :
Certaines options (OPT) sont réservées à des fonctions précises. Consultez la documentation métier du module concerné.
Performances sur AFCTFCY :
Pour les environnements avec 100+ sites, désactivez la mise à jour immédiate de AFCTFCY et utilisez l’utilitaire dédié.
5. Références Techniques Complémentaires
Paramètres Système :
ADMUSR : Compte super-administrateur (ignore les restrictions).
AUZFCT : Restreint les profils attribuables par un utilisateur.
Tables Clés :
AFCTPRF : Stocke les associations profil-fonction.
ADROIT : Gère les droits génériques (ex: accès aux menus).




Codes d'Accès et Rôles dans Sage X3


1.Introduction
Cette documentation technique fournit une description détaillée des mécanismes de gestion des codes d'accès et des rôles dans Sage X3. Ces fonctionnalités clés permettent un contrôle granulaire des autorisations et des restrictions d'accès au sein de l'ERP. Conçue pour les administrateurs système et les responsables sécurité, cette documentation couvre à la fois les aspects de configuration et les meilleures pratiques d'implémentation.


1. Gestion des Codes d'Accès (GESACS)


1.1 Introduction aux Codes d'Accès
Les codes d'accès constituent le niveau le plus fin de gestion des autorisations dans Sage X3. Ils permettent de contrôler précisément l'accès aux fonctions, données et processus spécifiques de l'ERP.


Objectifs :
- Comprendre l'architecture technique des codes d'accès
- Maîtriser la création et l'attribution des codes
- Implémenter des restrictions granulaires
- Automatiser la gestion des habilitations


Prérequis :
- Accès administrateur à Sage X3
- Connaissance des fonctions GESACS et GESARL
- Compréhension des besoins métiers en matière de sécurité


1.2 Structure Technique
Le système des codes d'accès s'appuie sur plusieurs tables clés :
- ADROIT : Table principale des droits
- ACS : Table des codes d'accès
- ACSOBJ : Associations codes d'accès/objets


1.3 Création et Configuration


1.3.1 Accès à la Fonction
Navigation : `Superviseur > Sécurité > Codes d'accès (GESACS)


1.3.2 Configuration de Base
Dans l'écran principal :
- CODACC : Code unique du droit (10 caractères max)
- DESACC : Description claire du droit
- TYPACC : Type de droit (Fonction, Objet, etc.)
- CATACC : Catégorie (Sécurité, Métier, etc.)


1.3.3 Attribution des Droits
Dans l'onglet "Attributions" :
- Associer le code aux objets concernés (écrans, transactions, etc.)
- Définir les niveaux d'accès :
  - Lecture (Consultation)
  - Ecriture (Modification)
  - Exécution (Validation/Traitements)


1.4 Bonnes Pratiques
- Utiliser une nomenclature cohérente (ex: "MODIF_FACT" pour modification des factures)
- Documenter chaque code d'accès créé
- Réviser trimestriellement les attributions
- Éviter la duplication des codes similaires


2. Gestion des Rôles (GESARL


2.1 Introduction aux Rôles
Les rôles permettent de regrouper des utilisateurs selon des critères organisationnels ou fonctionnels, et d'appliquer des filtres de données automatiques.


Cas d'usage typiques 
- Restriction par zone géographique
- Filtrage par département
- Accès spécifique à des types de documents


2.2 Structure Technique
Les rôles utilisent principalement :
- AROLES : Table des définitions de rôles
- AROLEUSR : Associations utilisateurs-rôles
- AROLEDAT : Restrictions de données par rôle


2.3 Création et Configuration**
2.3.1 Accès à la Fonction**
Navigation : `Superviseur > Sécurité > Rôles (GESARL)`


2.3.2 Configuration de Base
- CODROL : Code du rôle (5 caractères)
- INTROL : Intitulé du rôle
- TYPROL : Type (Hiérarchique, Fonctionnel, etc.)
- MODULE : Module principal concerné


2.3.3 Paramétrage Avancé
Dans l'onglet "Restrictions" :
- Définir les critères de filtrage des données
- Configurer les règles de visibilité
- Spécifier les exceptions éventuelles


2.4 Bonnes Pratiques
- Limiter le nombre de rôles aux besoins réels
- Documenter les règles de chaque rôle
- Tester systématiquement les filtres appliqués
- Auditer annuellement les attributions


3. Interaction entre Codes d'Accès et Rôles


3.1 Hiérarchie des Contrôles1. Les rôles filtrent d'abord les données visibles
2. Les codes d'accès déterminent ensuite les actions possibles


3.2 Exemple Intégré
Pour un responsable régional des ventes :

- Codes d'accès : 
  - "LECT_VENTE" (consultation)
  - "MODIF_VENTE" (modification limitée)
  - "VALID_BON" (validation des bons de commande < 10k€)


4. Dépannage et FAQ


4.1 Problèmes Courants
- Accès refusé malgré le code : Vérifier les conflits avec les rôles
- Données manquantes : Contrôler les filtres du rôle
- Performances : Optimiser les critères de filtrage
4.2 Outils de Diagnostic
- AFFDROIT : Visualisation des droits effectifs
- TRCADROIT : Traçabilité des modifications
- ANADROIT: Analyse des conflits potentiels

Création d’un profil fonctionnel
Étapes et champs à remplir
   1. Accéder à la fonction des profils fonctionnels
   * Navigation : Paramétrage > Utilisateurs > Profils fonctionnels (GESAFT).
   * Action : Cliquez sur "Nouveau" pour créer un profil.
   2. Saisir les informations générales
   * Champ : Code profil (PRFCOD)
   * Description : Identifiant unique (ex. : "RH_MANAGER").
   * Requis : Oui.
   * Étape : Saisi en premier lors de la création.
   * Champ : Description
   * Description : Nom descriptif (ex. : "Manager Ressources Humaines").
   * Requis : Non, mais recommandé.
   * Étape : Saisi après le code.
   3. Configurer l’onglet Accès
   * Champ : Toutes fonctions autorisées
   * Description : Cocher pour accorder un accès illimité (généralement pour les administrateurs).
   * Requis : Oui (choix entre coché ou non).
   * Étape : Configuré immédiatement après les informations générales.
   * Champ : Codes d’accès par défaut
   * Description : Liste des codes d’accès autorisés (ex. : ["CODE1", "CODE2"]).
   * Requis : Non, sauf si "Toutes fonctions autorisées" n’est pas coché.
   * Étape : Saisi si des restrictions spécifiques sont nécessaires.
   * Champ : Tous les codes d’accès
   * Description : Cocher pour inclure tous les codes sans liste exhaustive.
   * Requis : Non.
   * Étape : Alternative aux codes d’accès spécifiques.
   4. Configurer l’onglet Sites
   * Champ : Site par défaut par module
   * Description : Sélectionner un site par module (ex. : "PARIS" pour le module RH).
   * Requis : Non, uniquement pour les modules utilisés.
   * Étape : Configuré après l’onglet Accès.
   5. Configurer l’onglet Rapports
   * Champ : Impression différée
   * Description : Interdire l’impression en mode batch.
   * Requis : Non.
   * Étape : Configuré en dernier.
   * Champ : Filtre d’accès aux rapports
   * Description : Modèle avec jokers (ex. : "AB*") pour restreindre l’accès.

Création d’un profil de menu
Étapes et champs à remplir
   1. Accéder à la fonction des profils de menu
   * Navigation : Paramétrage > Utilisateurs > Profils de menu (GESAPN).
   * Action : Cliquez sur "Nouveau".
   2. Saisir les informations générales
   * Champ : Code profil
   * Description : Identifiant unique (ex. : "MENU_RH").
   * Requis : Oui.
   * Étape : Saisi en premier.
   * Champ : Description
   * Description : Nom descriptif (ex. : "Menu Ressources Humaines").
   * Requis : Non.
   * Étape : Saisi après le code.
   3. Configurer le menu de départ
   * Champ : Menu de départ
   * Description : Menu standard comme base (ex. : "ADMIN" ou "GENE").
   * Requis : Oui.
   * Étape : Sélectionné après les informations générales.
   4. Configurer les éléments de menu
   * Champ : Code fonction
   * Description : Code de la fonction à inclure (ex. : "GES_RH").
   * Requis : Oui.
   * Étape : Saisi pour chaque élément de menu.
   * Champ : Option
   * Description : Option spécifique (ex. : "EMP" pour employés).
   * Requis : Non.
   * Étape : Saisi après le code fonction.
   * Champ : Titre
   * Description : Texte affiché dans le menu (ex. : "Gestion RH").
   * Requis : Oui.
   * Étape : Saisi après l’option.
   5. Configurer les paramètres avancés
   * Champ : Type de profil
   * Description : Sélection dans le menu local 926 (limite les connexions simultanées).
   * Requis : Oui.
   * Étape : Configuré après les éléments de menu.
   * Champ : Valeur mémoire supplémentaire
   * Description : Mémoire en Ko pour l’instance (ex. : 1024).
   * Requis : Non.
   * Étape : Configuré en dernier.
   6. Enregistrer le profil
   * Action : Cliquez sur "Enregistrer".
